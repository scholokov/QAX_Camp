Опис курсу:

Практичний курс, що складається з основ програмування на Java, а також базових засад написання UI-автотестів для Web-додатків. Курс розроблено для спеціалістів, що вже мають досвід у мануальному тестуванні та хотіли б розвиватися в автоматизації, але поки не знають з чого почати.

Вимоги до студентів:

● Базові знання з тестування.
● Впевнений користувач ПК.
● Наявність власного ноутбуку або ПК для занять.
● Встановлене ПЗ: Intellij IDEA, Java 8+, Maven.

Після курсу ви зможете:

● Створювати UI-автотести на Java та стеку найактуальніших та найзатребуваніших технологій для автоматизації тестування Selenide/Selenium, Maven/Gradle, jUnit/TestNG
● Створювати власний фреймворк з нуля
● Налаштовувати VM у Google Cloud, підіймати на ній Jenkins та налаштовувати CI/CD систему з Docker та Selenoid
● Генерувати чіткі, зрозумілі та презентабельні звіти за допомогою Allure Framework
● Автоматизувати рутинні задачі, імітуючи дії користувача в браузері
● Претендувати на позицію Junior QA Automation Engineer
1. Вступ до автоматизації. Теорія по Java.
Що таке клас, метод, змінна. Модифікатори доступу. Написання першої програми.
2. Java
Цикли. Умовні оператори.
Статика.
Перетворення типів.
3. Java
Масиви та колекції.
4. Java
ООП: поліморфізм, наслідування, інкапсуляція, абстракція.
5. jUnit и TestNG
Основні анотації. Написання юніт тестів з використанням бібліотеки Assert.
6. Maven и Gradle
Створення проектів та підключення основних фреймворків та бібліотек
7. Git и GitHub
Генерація ключів. Команди clone, pull, commit, push та ін.
8. Error handlers. Loggers. Listeners
Обробка виключень. Створення та налаштування власного логера та лістенера
9. Selenide та Allure
Створення структури фреймворку, налаштування базових бібліотек та написання тестів. Розбір чітких та зрозумілих звітів.
10. X-Path та CSS
Практика написання селекторів. Параметризація тестів через TestNG.xml файл
11. Паттерн проектування автотестів "PageObject"
Написання тесту з вкиористанням паттерну PageObject
12. Явні и неявні очікування. Retry analyzer
Створення та налаштування власного ретрайеру для впавших тестів
13. Робота з Excel та CSV файлами. Data Provider
Написання методів для роботи з Excel та CSV файлами. Використання в текстах фабрики тестових даних Data Provider
14. Знайомство з Docker, Selenoid
Підключення докер технології до автотестів. Запуск браузеру в контейнері
15. Хмарні технології в автоматизації тестуванння
Налаштування VM та підняття Jenkins в Google cloud.
16. Хмарні технології в автоматизації тестуванння
Налаштування Continious Integration & Shedulers

Опис курсу:

Цель курса:

применить теоретические знания и получить практические навыки в тестировании, необходимые для профессионального роста и получения оффера

Минимальные требования:

теоретические знания основ тестирования

С чего состоит курс:

Продуктовые встречи: стендап, статусы по текущей реализации планирование будущего функционала груминг задач ретро проблемы QA встречи особенности тестирования текущие QA вопросы HR встречи резюме консультации по инструментам и ресурсам для поиска работы подготовка к прохождению собеседования технические собеседования Самостоятельная работа работа с макетами работа с требованиями составление тестовой документации непосредственно тестирование баг-репортинг Инструментарий с которым работаем: Jira - таск / баг треккер TM4J - Тест менеджмент инструмент Confluence - требования Figma - дизайн Telegram / Google Meet - коммуникация GitHub - репозиторий

Чем непосредственно заниматься:

выполнять функциональное тестировать веб приложения: сверяться с дизайном и требованиями использовать DevTools, HTML / CSS валидаторы выполнять нефункциональное тестирование: Usability Performance Разрабатывать тест план Разрабатывать тест кейсы / чеклисты работать с задачами в Jira красиво описывать баги, с последующей перепроверкой (перфекционизм - наше всё)

Как происходит работа:

работаем продуктовой командой, состоящей из: PO PM BA UI-UX DEV QA Разрабатываем несложные веб-приложения

Что получаешь по ходу:

возможность ошибаться и учиться на своих ошибках вносить и защищать свои предложения к продукт видеть как твои труды меняют продукт постоянно получать грамотный фидбек трудится и получать опыт в свое свободное время

Что НЕ даем:

сухую теорию, её можно прочитать в другом месте у нас нет “классических лекций”, все наши встречи направлены на получение определенного результата если ничего не делать, то и результата не будет. Потому у нас приходится “впрягаться” и “впахивать”

Длительность:

мы не ограничиваем в сроках, но заинтересованы в получении результата в минимальные сроки

Что получаешь на выходе:

практические навыки строчку в резюме, участия в разработке веб приложения рекомендации работодателю, в случае такого запроса предложения по работе от партнеров
